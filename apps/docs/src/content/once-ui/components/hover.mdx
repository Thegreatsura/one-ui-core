The `Hover` component wraps any element and displays an overlay on top when the user hovers over it or focuses on it. It's perfect for creating interactive cards with hover effects, image overlays, or any content that needs to reveal additional information on interaction.

  marginBottom="24"

## Accessibility

The component supports keyboard navigation. For non-focusable triggers, add `tabIndex={0}` to make the wrapper keyboard-accessible. If your trigger is already focusable (like a Card with `href`), use `tabIndex={-1}` to prevent duplicate focus stops.

<CodeBlock
  marginTop="16"
  marginBottom="24"
  preview={
    <Hover
      trigger={
        <Card padding="l" direction="column" gap="12" maxWidth={24} center align="center" radius="l" tabIndex={0}>
          <Text variant="heading-strong-m">Keyboard accessible</Text>
          <Text onBackground="neutral-weak" variant="body-default-s">
            Tab to this card to see the overlay
          </Text>
        </Card>
      }
      overlay={
        <Column fill padding="l" gap="12" background="surface" border="neutral-medium" center align="center" radius="l">
          <Text variant="heading-strong-l">
            Focused!
          </Text>
          <Text onBackground="neutral-weak" variant="body-default-s">
            Showing an overlay
          </Text>
        </Column>
      }
    />
  }
  codes={[
    {
      language: "tsx",
      label: "Accessibility",

`<Hover

By default, the hover effect is disabled on touch devices. You can set `touch="enable"` to allow hover interactions, or `touch="display"` to always show the overlay on touch devices.

## Interactive

By default, the overlay is not interactive and click events are passed through to the trigger. You can set `interactive` to make the overlay interactive.

<Feedback marginBottom="24" marginTop="16">Be careful when using interactive overlays, since they prevent click events from reaching the trigger.</Feedback>
